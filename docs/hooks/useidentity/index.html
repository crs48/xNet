useIdentity

  • How to access the current user’s identity
  • The difference between identity, did, and isAuthenticated

Section titled “Quick example”

Signature

Section titled “Signature”

Takes no parameters. Reads from the XNetProvider context.

Return value

Section titled “Return value”
FieldTypeDescription
identityIdentity | nullThe full Identity object from XNetProvider config, or null.
isAuthenticatedbooleantrue if either identity or authorDID is set.
didstring | nullThe user’s DID. Resolved from identity.did first, then authorDID.

Gotchas

Section titled “Gotchas”

isAuthenticated doesn’t mean full identity

isAuthenticated is true if either identity or authorDID was passed to XNetProvider. You can be “authenticated” with just a DID and no full Identity object — this is common during development.

Throws outside XNetProvider

Like all xNet hooks, useIdentity throws if called outside XNetProvider.
