xNet provides 15 property types organized into five categories.

## Basic types

### text

A string property with optional length and pattern constraints.

```ts
import { text } from '@xnetjs/data'

title: text({ required: true, maxLength: 500 })
draft: false
description: text({ placeholder: 'Enter a description...' })
slug: text({ pattern: /^[a-z0-9-]+$/ })
```

| Option        | Type      | Default | Description                   |
| ------------- | --------- | ------- | ----------------------------- |
| `required`    | `boolean` | `false` | Reject null/undefined values. |
| `minLength`   | `number`  | ‚Äî       | Minimum string length.        |
| `maxLength`   | `number`  | ‚Äî       | Maximum string length.        |
| `pattern`     | `RegExp`  | ‚Äî       | Regex pattern to match.       |
| `placeholder` | `string`  | ‚Äî       | UI hint (not enforced).       |

**Infers to:** `string`
**Coercion:** `String(value)`. Returns `null` for null/undefined.

---

### number

A numeric property with optional range and integer constraints.

```ts
import { number } from '@xnetjs/data'

priority: number({ min: 1, max: 5 })
score: number({ min: 0, integer: true })
```

| Option     | Type      | Default | Description               |
| ---------- | --------- | ------- | ------------------------- |
| `required` | `boolean` | `false` |                           |
| `min`      | `number`  | ‚Äî       | Minimum value.            |
| `max`      | `number`  | ‚Äî       | Maximum value.            |
| `integer`  | `boolean` | `false` | Round to nearest integer. |

**Infers to:** `number`
**Coercion:** `Number(value)`. If `integer`, applies `Math.round()`. Returns `null` for NaN.

---

### checkbox

A boolean property with an optional default.

```ts
import { checkbox } from '@xnetjs/data'

completed: checkbox({ default: false })
isPublic: checkbox({ default: true })
```

| Option     | Type      | Default | Description                       |
| ---------- | --------- | ------- | --------------------------------- |
| `required` | `boolean` | `false` |                                   |
| `default`  | `boolean` | ‚Äî       | Default value for null/undefined. |

**Infers to:** `boolean`
**Coercion:** `'true'`/`1` to `true`, `'false'`/`0` to `false`, else `Boolean(value)`.

---

## Temporal types

### date

A timestamp stored as Unix milliseconds.

```ts
import { date } from '@xnetjs/data'

dueDate: date()
scheduledAt: date({ includeTime: true })
```

| Option        | Type      | Default | Description                      |
| ------------- | --------- | ------- | -------------------------------- |
| `required`    | `boolean` | `false` |                                  |
| `includeTime` | `boolean` | `false` | UI hint for showing time picker. |

**Infers to:** `number` (Unix timestamp in ms)
**Coercion:** Accepts `number`, `Date` objects (`.getTime()`), ISO strings (`Date.parse()`). Returns `null` if unparseable.
**Validation:** Must be between 0 and year 3000 (`32503680000000`).

---

### dateRange

A range with a start and optional end date.

```ts
import { dateRange } from '@xnetjs/data'

timeline: dateRange()
sprint: dateRange({ includeTime: true })
```

| Option        | Type      | Default | Description |
| ------------- | --------- | ------- | ----------- |
| `required`    | `boolean` | `false` |             |
| `includeTime` | `boolean` | `false` | UI hint.    |

**Infers to:** `{ start: string; end?: string }` (ISO 8601 strings)
**Coercion:** Parses `start` and `end` through `new Date()` and converts to ISO strings.

## Selection types

### select

A single choice from a predefined list.

```ts
import { select } from '@xnetjs/data'

status: select({
  options: [
    { id: 'todo', name: 'To Do', color: '#6366f1' },
    { id: 'doing', name: 'Doing', color: '#f59e0b' },
    { id: 'done', name: 'Done', color: '#10b981' }
  ] as const,
  default: 'todo'
})
```

| Option     | Type             | Default | Description                                                 |
| ---------- | ---------------- | ------- | ----------------------------------------------------------- |
| `options`  | `SelectOption[]` | ‚Äî       | Available choices. Each has `id`, `name`, optional `color`. |
| `required` | `boolean`        | `false` |                                                             |
| `default`  | `string`         | ‚Äî       | Default option ID.                                          |

**Infers to:** Union of option IDs (e.g., `'todo' | 'doing' | 'done'`). Requires `as const` on the options array.
**Coercion:** Returns `default` for null/undefined. Tries case-insensitive name matching as fallback.

:::tip[Use `as const` for literal types]
Without `as const`, the type is just `string`. With it, you get exact literal types:

```ts
// Without as const ‚Üí string
select({ options: [{ id: 'todo', name: 'To Do' }] })

// With as const ‚Üí 'todo'
select({ options: [{ id: 'todo', name: 'To Do' }] as const })
```

:::

---

### multiSelect

Multiple choices from a predefined list.

```ts
import { multiSelect } from '@xnetjs/data'

tags: multiSelect({
  options: [
    { id: 'bug', name: 'Bug', color: '#ef4444' },
    { id: 'feature', name: 'Feature', color: '#3b82f6' },
    { id: 'docs', name: 'Docs', color: '#8b5cf6' }
  ] as const
})
```

| Option     | Type             | Default | Description         |
| ---------- | ---------------- | ------- | ------------------- |
| `options`  | `SelectOption[]` | ‚Äî       | Available choices.  |
| `required` | `boolean`        | `false` |                     |
| `default`  | `string[]`       | ‚Äî       | Default option IDs. |

**Infers to:** Union of option IDs as array (e.g., `('bug' | 'feature' | 'docs')[]`)
**Coercion:** Returns `default ?? []` for null/undefined. Coerces single string to `[value]`. Filters invalid entries.

## Reference types

### relation

A reference to another node by ID.

```ts
import { relation } from '@xnetjs/data'

project: relation({ target: 'xnet://my-app/Project' })
children: relation({ multiple: true })
```

| Option     | Type        | Default | Description                                |
| ---------- | ----------- | ------- | ------------------------------------------ |
| `target`   | `SchemaIRI` | ‚Äî       | Constrain to a specific schema (optional). |
| `required` | `boolean`   | `false` |                                            |
| `multiple` | `boolean`   | `false` | Allow array of references.                 |

**Infers to:** `string` (single) or `string[]` (multiple)
**Coercion:** Returns `null`/`[]` for null/undefined. Filters empty strings.

See [Relations](/docs/schemas/relations/) for patterns and temp ID resolution in transactions.

---

### person

A reference to a user by DID.

```ts
import { person } from '@xnetjs/data'

assignee: person()
collaborators: person({ multiple: true })
```

| Option     | Type      | Default | Description          |
| ---------- | --------- | ------- | -------------------- |
| `required` | `boolean` | `false` |                      |
| `multiple` | `boolean` | `false` | Allow array of DIDs. |

**Infers to:** `string` (DID) or `string[]` (multiple DIDs)
**Validation:** Must match DID pattern: `/^did:[a-z]+:[a-zA-Z0-9._:-]+$/`
**Coercion:** Returns `null`/`[]` for null/undefined. Filters invalid DIDs.

## Rich types

### url

A validated URL string.

```ts
import { url } from '@xnetjs/data'

website: url()
repository: url({ placeholder: 'https://github.com/...' })
```

| Option        | Type      | Default | Description |
| ------------- | --------- | ------- | ----------- |
| `required`    | `boolean` | `false` |             |
| `placeholder` | `string`  | ‚Äî       | UI hint.    |

**Infers to:** `string`
**Validation:** Must match `/^https?:\/\/.+/i`
**Coercion:** Trims whitespace. Auto-prepends `https://` if no protocol present.

---

### email

A validated email string.

```ts
import { email } from '@xnetjs/data'

contactEmail: email()
```

| Option        | Type      | Default | Description |
| ------------- | --------- | ------- | ----------- |
| `required`    | `boolean` | `false` |             |
| `placeholder` | `string`  | ‚Äî       | UI hint.    |

**Infers to:** `string`
**Validation:** Must match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
**Coercion:** Trims and lowercases.

---

### phone

A validated phone number string.

```ts
import { phone } from '@xnetjs/data'

contactPhone: phone()
```

| Option        | Type      | Default | Description |
| ------------- | --------- | ------- | ----------- |
| `required`    | `boolean` | `false` |             |
| `placeholder` | `string`  | ‚Äî       | UI hint.    |

**Infers to:** `string`
**Validation:** Must match `/^[+]?[(]?[0-9]{1,4}[)]?[-\s./0-9]*$/` with at least 7 digits.
**Coercion:** Trims, collapses whitespace. Preserves formatting.

---

### file

A content-addressed file reference.

```ts
import { file } from '@xnetjs/data'

avatar: file({ accept: ['image/*'], maxSize: 5 * 1024 * 1024 })
attachments: file({ multiple: true })
```

| Option     | Type       | Default | Description                                                  |
| ---------- | ---------- | ------- | ------------------------------------------------------------ |
| `required` | `boolean`  | `false` |                                                              |
| `multiple` | `boolean`  | `false` | Allow array of files.                                        |
| `accept`   | `string[]` | ‚Äî       | Allowed MIME types (e.g., `['image/*', 'application/pdf']`). |
| `maxSize`  | `number`   | ‚Äî       | Maximum file size in bytes.                                  |

**Infers to:** `FileRef` or `FileRef[]`

```ts
interface FileRef {
  cid: string // Content-addressed ID
  name: string // Original filename
  mimeType: string // MIME type
  size: number // File size in bytes
}
```

## Auto-populated types

:::note[Usually not needed]
Every node automatically gets `createdAt`, `updatedAt`, `createdBy`, and `updatedBy` as system fields ‚Äî you don't need to define them in your schema. These property builders exist for cases where you want timestamps or authorship to appear as **formal schema properties** (e.g., for generic table UIs that render columns from schema definitions). For most schemas, skip these entirely.
:::

These are read-only fields populated automatically by the system.

### created

Timestamp of when the node was created.

```ts
import { created } from '@xnetjs/data'

createdAt: created()
```

**Infers to:** `number` (Unix timestamp in ms)
**Behavior:** Set to `Date.now()` at creation time. Read-only afterward.

---

### updated

Timestamp of the last modification.

```ts
import { updated } from '@xnetjs/data'

updatedAt: updated()
```

**Infers to:** `number` (Unix timestamp in ms)
**Behavior:** Set to `Date.now()` on every update. Read-only.

---

### createdBy

DID of the user who created the node.

```ts
import { createdBy } from '@xnetjs/data'

author: createdBy()
```

**Infers to:** `string` (DID)
**Behavior:** Set to the author's DID at creation time. Read-only afterward.
