- How to access the current user's identity

  :::

## Quick example

```tsx
import { useIdentity } from '@xnet/react'

function UserBadge() {
  const { did, isAuthenticated } = useIdentity()

  if (!isAuthenticated) return <p>Not signed in</p>
  return <p>Signed in as {did?.slice(0, 20)}...</p>
}
```

## Signature

```ts
function useIdentity(): UseIdentityResult
```

Takes no parameters. Reads from the `XNetProvider` context.

## Return value

| Field             | Type               | Description                                                           |
| ----------------- | ------------------ | --------------------------------------------------------------------- |
| `identity`        | `Identity \| null` | The full Identity object from `XNetProvider` config, or `null`.       |
| `isAuthenticated` | `boolean`          | `true` if either `identity` or `authorDID` is set.                    |
| `did`             | `string \| null`   | The user's DID. Resolved from `identity.did` first, then `authorDID`. |

## Usage patterns

### Pass DID to useNode for presence

```tsx
const { did } = useIdentity()
const { presence } = useNode(PageSchema, pageId, { did: did ?? undefined })
```

### Conditional rendering

```tsx
const { isAuthenticated } = useIdentity()

if (!isAuthenticated) {
  return <OnboardingFlow />
}
return <AppContent />
```

## Gotchas

:::note[`isAuthenticated` doesn't mean full identity]
`isAuthenticated` is `true` if either `identity` or `authorDID` was passed to `XNetProvider`. You can be "authenticated" with just a DID and no full Identity object — this is common during development.
:::

:::caution[Throws outside XNetProvider]
Like all xNet hooks, `useIdentity` throws if called outside `XNetProvider`.
:::
