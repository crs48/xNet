## The three core hooks

| Hook                                       | When to use it                                   | Returns                                               |
| ------------------------------------------ | ------------------------------------------------ | ----------------------------------------------------- |
| [`useQuery`](/xNet/docs/hooks/usequery/)   | List or find nodes. Read-only.                   | `{ data, loading, error, reload }`                    |
| [`useMutate`](/xNet/docs/hooks/usemutate/) | Create, update, delete nodes.                    | `{ create, update, remove, restore, mutate }`         |
| [`useNode`](/xNet/docs/hooks/usenode/)     | Edit a single node with collaborative rich text. | `{ data, doc, update, syncStatus, remoteUsers, ... }` |

### When to use which

- **Listing data** (tables, lists, grids, search results) вЂ” `useQuery`
- **Reading a single node by ID** (detail views, cards) вЂ” `useQuery(schema, id)`
- **Creating or deleting** вЂ” `useMutate`
- **Updating properties** on a detail page вЂ” `useMutate().update` or `useNode().update`
- **Rich text editing** (TipTap, collaborative docs) вЂ” `useNode` (gives you a `Y.Doc`)
- **Showing who's online** (presence, cursors) вЂ” `useNode` (gives you `remoteUsers` and `awareness`)

### The decision tree

```
Do you need a Yjs document (rich text, canvas)?
  в”њв”Ђв”Ђ Yes в†’ useNode
  в””в”Ђв”Ђ No
        в”њв”Ђв”Ђ Reading data? в†’ useQuery
```

`useNode` is the only hook that manages a Yjs document and P2P sync connection. If you don't need collaborative editing, `useQuery` + `useMutate` are simpler and more efficient.

## Supporting hooks

| Hook                                           | Purpose                                    |
| ---------------------------------------------- | ------------------------------------------ |
| [`useIdentity`](/xNet/docs/hooks/useidentity/) | Get the current user's DID and auth status |
| `useComments`                                  | Thread-based comments on any node          |
| `useHistory`                                   | Change history for a node                  |
| `useUndo`                                      | Undo/redo for node changes                 |

## All hooks require `XNetProvider`

Every hook reads from a React context provided by `XNetProvider`. If you call a hook outside the provider, it throws:

```tsx
import { XNetProvider } from '@xnet/react'

function App() {
  return (
    <XNetProvider config={{ authorDID, signingKey }}>
      {/* All hooks work inside here */}
      <TaskList />
    </XNetProvider>
  )
}
```
